## 2. アプリの役割

### 正式名称

- **本アプリ:** 海外移住 資産寿命シミュレーター（簡易版）
- **兄弟アプリ:** タイ移住 月額生活費シミュレーター

### 本番 URL

- 本アプリ: https://life-runway-lite-liferunway.streamlit.app/
- 兄弟アプリ: https://life-spending-designer-lite-liferunway.streamlit.app/

### リポジトリ

- 本アプリ: https://github.com/life-runway/life-runway-lite

### 何をするアプリか

退職後・海外移住後の**資産が何歳までもつか**を見るための簡易シミュレーターです。

単なる資産残高計算機ではなく、以下をまとめて確認するためのアプリです。

- 退職後・海外移住後の資産寿命
- 海外生活中の月額生活費
- 公的年金
- 企業年金・個人年金
- その他収入
- 運用利回り
- インフレ
- 日本帰国の有無
- 帰国後生活費
- 持ち家売却・海外住まい売却などの資産イベント

### 海外移住全般向け / タイ専用ではない

本アプリは**タイに限らず、海外移住全般**に使える資産寿命シミュレーターです。

一方、兄弟アプリ「タイ移住 月額生活費シミュレーター」はタイ生活費に特化しています。

**想定する連携:**

1. タイ移住を想定する場合、兄弟アプリで月額生活費の目安を試算する
2. その金額を、本アプリの「海外生活中の月額生活費（万円）」に入力する
3. 資産寿命が短い場合は、生活費・帰国年齢・年金・売却などの条件を見直す

他国移住の場合は、ユーザーが自分で生活費を見積もって入力します。

## 3. 入力金額の基本方針

- **入力はすべて日本円ベース**
- **金額単位は原則「万円」**（UI 上 `%` の利回り・インフレ率を除く）
- **生活費は将来の名目金額ではなく、現在の物価感覚で入力する**
- **生活費の将来増加は、入力されたインフレ率をもとにアプリ内部で反映する**

特に重要な点:

- 海外生活中の月額生活費も、日本帰国後の月額生活費も、**現在の物価感覚**で入力する
- 30 年後の名目生活費をユーザーが自分で計算して入力する仕様**ではない**
- 海外の生活費・不動産売却額も、日本円に換算して入力する

## 4. 資産寿命の目標年齢

### 意味

「資産寿命の目標年齢」は、**何歳時点まで資産が残っているか**を判定するための目標年齢です。

- モンテカルロの**成功率**（資産が残る可能性）の判定基準
- 結果コメントの出し分け基準
- 目標年齢時点の中央値残高・不足額表示の基準

### 寿命予測ではない

これは**寿命予測**ではありません。試算上、「その年齢まで資産を持たせたい」という**目標年齢**です。

### UI 設定（`app.py`）

| 項目 | 値 |
|------|-----|
| min | 80 |
| max | 100 |
| default | 90 |
| step | 1 |
| key | `target_age` |

## 6. 海外生活中の月額生活費

### 計算上の扱い

- **月額で入力**し、計算時に `月額 × 12` で年額支出に変換
- 各年齢の支出には **インフレ率を反映**（後述）
- 入力値は**現在の物価感覚**での金額

計算式（`simulator.py`）:

```text
annual_expense = monthly_cost_at_age × 12 × (1 + inflation_rate) ^ (age - start_age)
```

| 項目 | 値 |
|------|-----|
| min | 5 |
| max | 100 |
| default | 30 |
| step | 1 |
| key | `overseas_monthly_cost` |

### 兄弟アプリとの接続点

本項目は、兄弟アプリとの**接続点**です。

- **タイ移住を想定している場合:** 「タイ移住 月額生活費シミュレーター」で月額生活費の目安を試算し、その金額を本欄に入力する流れを想定
- **本アプリ自体はタイ専用ではない:** 他国移住の生活費も入力できる

UI 上の案内文（`app.py`）:

```text
タイ移住を想定している場合は、「タイ移住 月額生活費シミュレーター」で月額生活費の目安を確認できます。
```

リンク先: https://life-spending-designer-lite-liferunway.streamlit.app/

## 7. 日本帰国の有無・帰国年齢・帰国後生活費

### 基本

- **帰国予定なし:** 海外生活費を継続
- **帰国予定あり:** 指定した帰国年齢以降、日本帰国後の月額生活費を使用（`monthly_cost_at_age`）

### 日本帰国後の月額生活費

- **現在の日本の物価感覚**で入力
- 帰国時点までの物価上昇は、インフレ率で自動反映（海外生活費と同じ `start_age` 基準の複利）

| 項目 | 値 |
|------|-----|
| min | 5 |
| max | 100 |
| default | 30 |
| step | 1 |
| key | `japan_monthly_cost` |

### 帰国年齢

- **海外生活開始年齢より後**である必要がある
- UI 上、`min = start_age + 1` の**動的最小値**で制御
- 初期値は `min(max(80, start_age + 1), 100)`
- `start_age` 変更時は `_clamp_session_value` で `return_age` を補正

| 項目 | 値 |
|------|-----|
| min | `start_age + 1`（動的） |
| max | 100 |
| step | 1 |
| key | `return_age` |

## 9. 公的年金の扱い

### 入力の意味

- **「65 歳時点の公的年金年額（万円）」** を入力する
- **繰上げ・繰下げ後の実際の受取額を入力する仕様ではない**
- 受給開始年齢（60〜70 歳）を選び、その年齢に応じた係数を **1 回だけ** 掛ける（`get_public_pension_annual`）

### 終身収入

公的年金は受給開始後、**100 歳まで毎年加算**される終身収入として扱う（`end_age=None`）。

### 繰上げ・繰下げ係数（`PENSION_COEFFICIENTS`）

65 歳時点年額を 100% とした簡易係数:

| 受給開始年齢 | 係数 |
|-------------|------|
| 60 歳 | 76.0% |
| 61 歳 | 80.8% |
| 62 歳 | 85.6% |
| 63 歳 | 90.4% |
| 64 歳 | 95.2% |
| 65 歳 | 100% |
| 66 歳 | 108.4% |
| 67 歳 | 116.8% |
| 68 歳 | 125.2% |
| 69 歳 | 133.6% |
| 70 歳 | 142.0% |

実際の年額 = `public_pension_at_65 × 係数`

### UI 設定

**65 歳時点の公的年金年額:**

| 項目 | 値 |
|------|-----|
| min | 0 |
| max | 400 |
| default | 200 |
| step | 10 |
| key | `public_pension_at_65` |

**受給開始年齢:**

| 項目 | 値 |
|------|-----|
| min | 60 |
| max | 70 |
| default | 65 |
| step | 1 |
| key | `public_pension_start_age` |

### 注意点（二重適用の誤り）

**70 歳開始時の実受取見込額**（例: 284 万円）を入力し、さらに受給開始年齢を 70 歳にすると、繰下げ係数が**二重に**かかる（284 × 1.42 ≒ 403 万円）ため**誤り**になります。

**必ず 65 歳時点の年額を入力**してください。help 文言でもこの点を案内しています。

本簡易版では、公的年金の税金・社会保険料は個別計算しません。

## 10. 企業年金・個人年金などの扱い

### 位置づけ

- **ある人だけ**が入力する追加収入
- 一般利用者全員にある項目ではなく、expander「企業年金・個人年金などがある場合」内の任意入力

### 入力仕様

- **年額（万円）** で入力
- **最大 2 本**まで（`render_income_streams`）
- **開始年齢**（受給開始年齢）を指定
- **終了**は「終身」または「年齢指定」

| 項目 | 年額 | 開始年齢 | 終了年齢 |
|------|------|----------|----------|
| min | 0 | 55 | `s_age`（動的） |
| max | 500 | 100 | 100 |
| default | 0 | 65 | `s_age` |
| step | 10 | 1 | 1 |
| key | `corp_pension_amount_{0,1}` | `corp_pension_start_{0,1}` | `corp_pension_end_{0,1}` |

### 終身の意味（重要）

- **「終身」を選んだ場合、100 歳まで毎年加算する**
- 「終身」は現実の寿命を予測する意味**ではない**
- シミュレーション終端年齢（`END_AGE = 100`）まで継続する収入として扱う**実装上の意味**

有期の企業年金・個人年金の場合は、**必ず終了年齢を指定**する必要があります。デフォルトが「終身」のため、有期収入を入れる際は「年齢指定」を選ぶこと。

### 未入力扱い

**年額が 0 のものは収入として数えない**（未入力扱い）。

## 11. その他収入の扱い

### 位置づけ

企業年金・個人年金**以外**の継続的な収入を入力する欄。expander「その他収入がある場合」内。

### 入力仕様

企業年金と**同じロジック**（`IncomeStream` / `income_at_age`）。

| key プレフィックス | `other_income_` |
|-------------------|-----------------|
| 年額 key | `other_income_amount_{0,1}` |
| 開始年齢 key | `other_income_start_{0,1}` |
| 終了年齢 key | `other_income_end_{0,1}` |
| 開始年齢 default | 60（企業年金は 65） |

### 入力例

- 賃貸収入
- 副業収入
- 家族からの継続的支援
- その他の定期収入

### 二重計上に注意

配当収入や投資運用益は、**運用利回りと二重計上**になる可能性があります。運用利回りで既に反映している前提なら、その他収入に含めない方がよい場合があります。

### 終身

企業年金と同様、**「終身」は 100 歳まで毎年加算**されます。

## 12. 運用利回りの扱い

### 入力

- UI 上は**年率 %**（0.0〜5.0、default 2.0、step 0.5）
- 内部では小数（2% → 0.02）に変換（`return_rate_pct / 100.0`）
- key: `return_rate_pct`（float、`decimals=1`）

### 計算順序（`simulate_path`）

各年齢について、以下の順で処理します。

```text
1. 帰国年齢の年初: 海外の住まい売却額を加算（該当する場合のみ）
2. 年初資産に運用利回りを反映: balance × (1 + yearly_return)
3. その年の収入（年金・その他）を加算
4. インフレ反映後の生活費を控除
5. 年末資産を記録（マイナスでもそのまま記録）
```

### 簡易試算であること

税金・手数料・口座制約・個別銘柄リスクなどを厳密に再現するもの**ではない**。

## 13. インフレ率の扱い

### 反映対象

- **生活費のみ**に反映
- **収入側にはインフレ率を掛けない**

### 計算方法

- `start_age` を基準年とし、年齢が進むごとに複利で増加:

```text
inflation_factor = (1 + inflation_rate) ^ (age - start_age)
annual_expense = monthly_cost × 12 × inflation_factor
```

### UI 設定

| 項目 | 値 |
|------|-----|
| min | 0.0 |
| max | 5.0 |
| default | 2.0 |
| step | 0.5 |
| key | `inflation_rate_pct` |

### 設計上の含意

長期試算では、**生活費は物価上昇を考慮する一方、年金・その他収入は名目固定**として扱います。支出側がやや保守的に働く場合があります。

## 14. モンテカルロ試算と参考シナリオ

### 2 種類の試算

| 種類 | 内容 |
|------|------|
| **参考シナリオ** | 想定利回り・インフレ率が毎年そのまま続く固定的な試算（`reference_scenario`） |
| **モンテカルロ試算** | 利回りにばらつきを与えた 1,000 通りの試算（`run_simulation`） |

### 計算期間

- `start_age` から **`END_AGE = 100` 歳**まで

### モンテカルロ定数（`simulator.py`）

| 定数 | 値 | 意味 |
|------|-----|------|
| `MC_TRIALS` | 1000 | 試行回数（固定） |
| `RETURN_STD` | 0.08 | 各年利回りに乗せる正規分布の標準偏差（8%） |
| `RETURN_CLIP` | (-0.5, 0.5) | 利回りの上下限（-50%〜+50%） |
| `RANDOM_SEED` | 42 | 乱数シード（同じ入力なら同じ結果） |

各試行・各年の利回り:

```text
yearly_return = clip(return_rate + Normal(0, RETURN_STD), RETURN_CLIP)
```

### 結果の集計

| 出力 | 意味 |
|------|------|
| `median_path` | 標準ケース（中央値、50 パーセンタイル） |
| `low_path` | 厳しめケース（下位 10%） |
| `high_path` | 良好ケース（上位 10%） |
| `reference_path` | 参考シナリオ |

## 15. 資産寿命・成功率の判定

### 資産が尽きる目安（`depletion_age`）

- **標準ケース（中央値）** で、初めて残高がマイナスになった年齢
- マイナスにならなければ「100 歳まで資産が残る見込み」

### 資産が残る可能性（`survival_probability`）

- **目標年齢（`target_age`）時点**で、残高 ≥ 0 の試行割合
- 1,000 試行中何割が「目標年齢まで資産が残ったか」

### 目標年齢時点の中央値残高（`balance_at_target`）

- 標準ケース（中央値）における目標年齢時点の資産残高
- マイナスの場合は不足額として赤字表示

### 読み方の注意

- **中央値残高がマイナスでも**、好調試行が一部あれば成功率はゼロにならない場合がある
- **良好ケース（上位 10%）** は、通常より楽観的に見えることがある
- 成功率は**目標年齢時点のみ**を見ており、それ以降の枯渇は別途 `depletion_age` で確認する

## 16. コメント生成の考え方

`comments.py` は AI API を使わず、**ルールベース**でコメントを生成します。

### 3 部構成

1. **総合コメント** — 成功率と中央値残高に応じた読み解き
2. **要因コメント** — 入力条件から検出した影響要因（最大 2 件）
3. **次に試す変更案** — 改善方向・保守的確認の提案（最大 3 件）

### トーン方針

- **投資助言・断定にならない**
- 避ける表現: 「移住できます」「退職して大丈夫です」「安心です（断定）」「投資を増やすべきです」など
- 使う表現: 「確認してみてください」「比較してみてください」「不足が出る可能性があります」など
- 不安をあおりすぎず、過度に安心させすぎないバランス

### 総合コメントの分岐（成功率）

| 成功率 | おおむねのトーン |
|--------|-----------------|
| ≥ 90% | かなり高め。ただし厳しめケースも確認を促す |
| ≥ 80% | 比較的高め。余裕の確認を促す |
| ≥ 60% | 一定程度あるが余裕は限定的。改善余地の確認を促す |
| ≥ 40% | 50% 前後。改善方向の前提変更を促す |
| ≥ 20% | 不足の可能性が高め。生活費・開始年齢・年金開始前の期間を示唆 |
| < 20% | かなり厳しい。複数前提の見直しを促す |

### 中央値残高の補足（`_balance_note`）

| 残高（万円） | 判定 |
|-------------|------|
| < 0 | 総合コメント側で不足として扱う（補足なし） |
| 0 以上 < 500（`SLIGHT_SURPLUS`） | かなりギリギリ |
| 500 以上 < 1,000（`LIMITED_SURPLUS`） | 余裕は限定的 |
| ≥ 1,000 | 補足なし |

### 要因コメントの検出条件（`_detect_factors`）

| キー | 条件 |
|------|------|
| `pension_gap` | 公的年金開始年齢 − 海外生活開始年齢 ≥ 3 |
| `high_living` | 海外月額生活費 ≥ 35 万円 |
| `high_inflation` | インフレ率 ≥ 3% |
| `return_plan` | 日本帰国予定あり |
| `japan_home_sale` | 日本の持ち家売却あり・手取り > 0 |
| `overseas_home_sale` | 海外住まい売却あり・手取り > 0 |
| `other_income` | その他収入 > 0 |
| `generic` | 上記いずれも該当しない場合 |

**注意:** 企業年金・個人年金入力に関する要因コメントは現状なし。

### 次に試す変更案（`_next_try_comments`）

- 成功率 ≥ 80%: 保守的確認中心（最大 2 件）
- 成功率 ≥ 60%: 改善案と保守的確認のバランス（最大 3 件）
- 成功率 < 60%: 改善方向を優先（最大 3 件）
- 生活費引き下げ、開始年齢の遅延、インフレ率・利回りのストレステストなどを提案

## 17. 兄弟アプリとの連携

### 役割分担

| アプリ | 役割 |
|--------|------|
| **タイ移住 月額生活費シミュレーター** | タイで暮らす場合の月額生活費を見える化 |
| **海外移住 資産寿命シミュレーター（簡易版）** | その月額生活費を使って資産寿命を見る |

### 想定する使い方

1. 「タイ移住 月額生活費シミュレーター」で、タイ生活の月額生活費を試算する
2. その金額を、本アプリの「海外生活中の月額生活費（万円）」に入力する
3. 資産寿命が短い場合は、生活費・帰国年齢・年金・資産売却などの条件を見直す
4. 必要に応じて、兄弟アプリに戻って生活費を再調整する

### 重要な区別

- **本アプリはタイ専用ではない**
- 兄弟アプリへの導線は、**タイ移住を想定している人向けの補助導線**
- 案内文は「タイ移住を想定している場合は」と限定している（`app.py` の `st.caption`）

## 18. Streamlit UI 実装上の注意

### −／＋ ステッパーボタン

全スライダーに `slider_with_steppers` で −／＋ ボタンを付けている。

- **スマホ操作性改善**のための UI 補助
- **計算ロジックには影響しない**
- 既存の `step` 単位で増減
- `min` / `max` を超えないようクランプ（`_step_session_value`）
- float 型（利回り・インフレ率）は `decimals=1` で丸め

### 動的 min の補正

| スライダー | 補正 |
|-----------|------|
| `return_age` | `_clamp_session_value("return_age", start_age + 1, 100, default)` |
| `{prefix}_end_{i}` | `_clamp_session_value(end_key, s_age, 100, s_age)` |

### 計算ボタン方式

- スライダー操作のたびに再計算**しない**
- 「計算する」ボタン押下時のみ `run_simulation` を実行
- 結果は `st.session_state` に保存（`sim_input`, `result`, `comment_set`）

### Streamlit の特性

- ボタン操作ごとに**スクリプト全体が再実行**される
- 通常のスマホアプリのように数値だけが瞬時に変わるわけ**ではない**
- 反応が少し鈍く感じることは Streamlit の特性

### ステッパー CSS

- `st.container(key="stepper_{key}")` + `[class*="st-key-stepper_"]` スコープの CSS
- 「計算する」primary ボタンとはセレクタを分離

## 19. Streamlit Cloud 公開時の注意

- Streamlit Community Cloud では、**push 後の再デプロイに数分かかる**ことがある
- 反映直後は**旧版に見える**場合がある
- 公開 URL 確認時は数分待って再読み込みする

### session_state 変更時

関数引数や `session_state` の key を変更する場合、既存セッションと不整合にならないよう、**デフォルト値・クランプ処理**を意識する。
